# 193. Volumes (磁碟區與掛載)

## 1. 🏷️ 課程定位

- **影片標題：** 193. Volumes (磁碟區與掛載)

## 2. 📌 核心概念摘要
Kubernetes 中的 Volume (磁碟區) 是為了解決容器檔案系統「短暫性 (Ephemeral)」的先天缺陷。透過在 Pod 中宣告 Volume 並將其掛載 (Mount) 至容器內部的指定目錄，我們可以讓資料的生命週期脫離單一容器的生與死，確保容器重啟時資料不會遺失，甚至能讓同一個 Pod 內的多個容器共享同一份資料。

## 4. 🔑 知識點擷取 (Detailed Notes)
這張 YAML 截圖雖然還沒寫完（缺少了 `volumeMounts`），但它點出了 K8s 儲存設定的核心邏輯——「兩段式設定」：

- **第一段：宣告儲存來源 (`spec.volumes`)：**
  定義這個 Pod 擁有哪些儲存資源。必須指定一個 `name` (如 `data-volume`) 作為內部識別碼，並宣告其底層 `type` (如 `hostPath`, `emptyDir`, `nfs` 等)。

- **第二段：容器掛載點 (`spec.containers[].volumeMounts`)：**
  在指定的容器內，透過引用剛剛宣告的 `name`，並指定 `mountPath` (例如 `/opt`)，將儲存空間正式接通到容器的檔案系統中。

- **⚠️ 致命限制條件 (hostPath 的實務陷阱)：**
  `hostPath` 等同於 Docker 的 Bind Mount，它直接將 Pod 綁死在「當前這台 Node 的特定目錄」上。
  - **失去高可用性：** 如果這台 Node 故障，Pod 被 Kubernetes 重新調度 (Reschedule) 到 Node 2，Node 2 上如果沒有 `/data` 這個目錄，Pod 將會啟動失敗；即使有，裡面的資料也跟 Node 1 完全不同。
  - **Type 驗證：** YAML 中的 `type: Directory` 是一個嚴格檢查條件，代表「啟動前，宿主機上必須已經存在這個目錄」，否則 Kubelet 會拒絕掛載。

## 5. 💻 CKA 必備實作指令 (Imperative Commands)
在 CKA 考場中，我們無法用一行 `kubectl run` 就變出包含 Volume 的 Pod。最穩定的做法是「先產骨架，再用 Vim 編輯」。

```bash
# 💡 CKA 考試技巧：先產生乾淨的 Pod YAML 骨架
kubectl run random-number-generator --image=alpine --dry-run=client -o yaml --command -- sh -c "shuf -i 0-100 -n 1 >> /opt/number.out" > pod.yaml

# 💡 考場救命指令：考場上若忘記 Volume 到底有哪些參數，絕對不要死背，直接用 explain 查！
# 查詢如何設定 hostPath
kubectl explain pod.spec.volumes.hostPath

# 查詢如何設定 volumeMounts
kubectl explain pod.spec.containers.volumeMounts
```

## 6. 🚀 CKA 考試延伸與 Troubleshooting
### 🎯 考試情境預測：
- **必考題型 (Multi-Container 共享資料)：** 考題極高機率會要求你建立一個包含兩個容器的 Pod (例如一個 Nginx 負責讀取，一個 Busybox 負責寫入)，並要求你建立一個 `emptyDir` 類型的 Volume，讓這兩個容器掛載到各自的目錄，達成資料共享。
- **節點日誌收集：** 要求你在特定 Node 上跑一個 Pod，並透過 `hostPath` 掛載 Node 的 `/var/log` 目錄以讀取系統日誌。

### 🛑 避坑指南 (縮排與名稱對應地獄)：
- **層級搞錯：** `volumes` 陣列是與 `containers` 平行的（都在 `spec` 正下方）；而 `volumeMounts` 是包在單一 `container` 裡面的屬性。縮排錯一格，Apply 就會報錯。
- **名稱不匹配：** `volumeMounts` 中的 `name` 必須與 `volumes` 中的 `name` 字母大小寫完全一致。

### 🔧 Troubleshooting：
- **現象：執行 `kubectl apply` 後，Pod 狀態一直卡在 `ContainerCreating`。**
  - **排查 3 步驟：**
    1. 執行 `kubectl describe pod <pod-name>` 查看最下方的 Events。
    2. 如果看到 `hostPath type check failed: /data is not a directory`，代表你設定了 `type: Directory`，但 Node 上根本沒有這個資料夾（解法：改用 `DirectoryOrCreate` 或手動去 Node 建資料夾）。
    3. 如果看到 `MountVolume.SetUp failed`，通常是 Kubelet 權限不足，或是掛載路徑被系統的 SELinux 擋住了。
