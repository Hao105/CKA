# 2. Certification (CKA 認證考試全解析)

## 🎯 核心觀念

- **100% 實戰導向**：CKA 是一場純粹的實作測驗 (Hands-on Performance-based)，沒有選擇題或申論題。你必須在 2 小時內於真實的 Linux 終端機環境中，精準完成約 15 到 20 道叢集建置、資源部署與故障排查任務。
- **考試基礎規格**：
  - **及格門檻**：總分 100 分，及格線為 66%。
  - **考試環境**：全程於 PSI 瀏覽器內建的 Ubuntu 終端機進行，並由監考官遠端視訊與螢幕分享監考。
- **「Open Book」的藝術**：考試允許開啟一個額外分頁查閱 [Kubernetes 官方文檔](https://kubernetes.io/docs/)。實務上你不需要死背長篇大論的 YAML 結構，但你必須具備**極強的關鍵字搜尋能力**，能在 10 秒內找到官方範例並複製修改。
- **多叢集切換陷阱 (Context Switching)**：考場內並不是只有一個 K8s 叢集！考題會散落在 4 到 6 個不同的叢集中。每道題目最上方都會提供切換叢集的指令，若忘了切換，不管做得多完美都等同 0 分。

## 💻 必考實戰指令 (保命防呆起手式)

這些指令不是用來直接解題，而是確保你在正確的戰場上戰鬥，並大幅提升解題速度：

```bash
# 1. 🛑 考場第一鐵律：每做一題前，務必複製並執行題目提供的切換指令
kubectl config use-context k8s-dev

# 2. 👁️ 確認自己目前身處哪個叢集 (防呆機制)
kubectl config current-context

# 3. 🚀 考場加速神器：設定別名與自動補全 (Autocomplete)
# 通常現代考場已經預設開啟，若沒有，這兩行能為你省下極大的打字時間
alias k=kubectl
source <(kubectl completion bash)
complete -o default -F __start_kubectl k
```

> [!CAUTION]
> **致命失誤：做錯叢集 (Wrong Context)**
> 這是無數考生落榜的頭號死因！如果您忘記在解第 5 題前執行該題專屬的 `use-context` 指令，不小心把資源建在第 4 題的叢集裡，該題直接 0 分，系統評分腳本絕對不會幫您尋找檔案。

> [!WARNING]
> **SSH 迷航陷阱**
> 在 Troubleshooting 題型中，題目常會要求你 `ssh` 進入某台 Worker Node 去重啟 `kubelet` 等底層元件。修復完成後，**務必記得輸入 `exit` 退出該節點**，回到預設的 Jumpbox 跳板機，否則接下來的所有 `kubectl` 指令都會因缺乏憑證而報錯。

> [!TIP]
> **配分戰略與排版排錯 (Troubleshooting)**
> - **配分戰略**：每題都會標示權重（如 4% 或 12%）。遇到配分極低且困難的題目，別卡超過 5 分鐘！善用考場系統的「Flag（標記）」功能跳過，優先搶下高配分大題，確保總分突破 66%。
> - **縮排跑掉解法**：在終端機使用 `vim` 貼上官方 YAML 時，縮排常會大亂。貼上前請先輸入 `:set paste` 進入貼上模式，或者改用 `nano` 編輯器，可大幅降低這類低級錯誤的發生率。

## 📝 YAML 骨架範例 (Imperative 轉 Declarative 技巧)

考場上切忌從頭手刻 YAML！你必須具備利用 Imperative 指令快速生成 YAML 骨架的能力，再以 `vim` 進行微調：

```bash
# 考場必殺技：利用 --dry-run 結合輸出導入，快速產生無誤的基礎 YAML 骨架
kubectl run nginx-pod --image=nginx --dry-run=client -o yaml > pod.yaml
```

**快速生成的 pod.yaml 骨架**：
```yaml
apiVersion: v1
kind: Pod
metadata:
  creationTimestamp: null
  labels:
    run: nginx-pod
  name: nginx-pod
spec:
  containers:
  - image: nginx
    name: nginx-pod
    resources: {}
  dnsPolicy: ClusterFirst
  restartPolicy: Always
status: {}
```
*取得完美的語法骨架後，只需使用 `vim pod.yaml` 編輯檔案，將官網查到的特例配置（如 Volume, SecurityContext）貼入即可。*

**防禦 SOP：**
1. **確認位置**：確保你現在身處預設的 Jumpbox（跳板機）。如果你剛才的題目有 `ssh` 進入其他 Node，請務必先執行 `exit` 回來。
2. **切換叢集**：仔細查看第 4 題最上方給定的叢集切換指令（例如 `kubectl config use-context k8s-trouble`），並複製貼上執行。
3. **雙重驗證**：(非必須但推薦) 執行 `kubectl config current-context` 再次確認大腦已經正確切換到該目標叢集，然後再開始閱讀題目與解題。
