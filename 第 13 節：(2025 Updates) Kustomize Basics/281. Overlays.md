# 281. Overlays (環境隔離與疊加層架構)

## 🎯 核心觀念

- **「千層派」式架構**：Overlays 是 Kustomize 實現「環境隔離」的終極架構。底層是永遠不變的 Base（基礎層），上層則是針對不同環境（Dev, Prod）層層堆疊的補丁（Overlays），像千層派一樣疊加出最終符合該環境的 YAML 清單，徹底終結了以往複製貼上多份 YAML 的維護惡夢。
- **Base (基礎層) 的唯讀原則**：Base 目錄存放最基礎、跨環境皆通用的 Kubernetes 核心資源（如相同的 Service Port 或 Volume 掛載點）。Base 應該保持「乾淨且唯讀」，絕不可參雜特定環境的雜音（例如把 replicas 寫死成生產環境的數字）。
- **Overlay (疊加層) 的增量哲學**：Overlay 代表具體部署環境。它透過 `resources:` 相對路徑呼叫 Base，且**僅存放自己環境「與眾不同」的配置**（例如 Dev 只開 1 個副本，Prod 開 10 個副本）。
- **微服務級別的精密控制**：不僅全域有 Base/Overlay，底層的微服務元件（如 `db` 與 `api`）也能擁有獨立的 `kustomization.yaml`。這允許維運人員對單一組件進行精密的環境配置切換，而不影響周邊元件。

## 💻 必考實戰指令

```bash
# 1. 👁️ 考場神技：在部署前，先行對比特定環境與目前叢集的「實際差異」
# 這能讓你一眼看出此環境補丁（如副本數或 Image 替換）是否會如預期生效
kubectl diff -k k8s/overlays/dev/

# 2. 📝 純文字渲染：驗證正式環境 (Prod) 是否正確將 Base 與生產補丁融合
# 檢查輸出的 YAML 中，Prod 的專屬欄位（如資源限制、安全設定）是否正確注入
kubectl kustomize k8s/overlays/prod/

# 3. 🚀 一鍵將特定環境打包並部署到叢集
# Kustomize 會自動走訪相對路徑搬取 base，並與 overlays/stg 合併後送出
kubectl apply -k k8s/overlays/stg/
```

> [!CAUTION]
> **考場陷阱：相對路徑的「點點」慘案**
> Kustomize 強烈依賴相對路徑。在多層級目錄（如 `k8s/overlays/dev/db/`）中，要一路上溯回 `k8s/base/db/`，相對路徑必須精準計算（例如 `../../../../base/db`）。少寫一個 `../` 就會引發找不到資源的致命錯誤！

> [!IMPORTANT]
> **遺漏的主控檔：無所不在的 kustomization.yaml**
> 無論是 base、overlays/dev 還是底下的子目錄 db，只要它想參與 Kustomize 的編排，該目錄下**必須**存在一個合法的 `kustomization.yaml`。如果只放了 patch 檔案卻沒寫主控檔，Kustomize Build 會直接無視該目錄。

## 📝 YAML 骨架範例

**1. Base 基礎層 (`base/kustomization.yaml`)**
```yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization

resources:
  - deployment.yaml
  - service.yaml
```

**2. Prod 疊加層 (`overlays/prod/kustomization.yaml`)**
```yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization

# 繼承不變的 Base 資源
resources:
  - ../../base

# 套用僅屬於 Prod 環境的增量補丁
patches:
  - path: prod-replica-patch.yaml
```

> [!TIP]
> **Troubleshooting 技巧：Isolated Root 報錯解析**
> **錯誤**：執行 `kubectl apply -k` 時出現 `error: ... loader: inside isolated root`
> **原因排查**：這代表你的 `resources:` 裡面寫了**絕對路徑**（如 `/root/k8s/base`）或是跳出了 Kustomize 規範的根目錄封裝範圍。請立刻將其修改為標準的相對路徑（如 `../../base`）。

## 🧠 自我測驗

<details>
<summary>在考場上，考官提供了一個現成的目錄結構 <code>base</code> 與 <code>overlays/production</code>。要求你編輯 <code>overlays/production/kustomization.yaml</code>，使其正確繼承 base 資源，並套用同目錄下的 <code>replica-patch.yaml</code>。你會如何撰寫該檔案？</summary>

在 `overlays/production/kustomization.yaml` 中，必須透過 `resources` 使用相對路徑引入 base，並在 `patches` 中指定該補丁檔名：

```yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization

resources:
  - ../../base

patches:
  - path: replica-patch.yaml
```

完成編輯後，強烈建議先執行 `kubectl kustomize overlays/production/` 預覽合併結果，確認無誤後再以 `kubectl apply -k overlays/production/` 將其部署到叢集中。
</details>
