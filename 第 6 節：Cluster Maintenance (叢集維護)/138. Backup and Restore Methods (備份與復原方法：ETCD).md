# 138. Backup and Restore Methods (備份與復原方法：ETCD)

## 1. 🏷️ 課程定位
- **章節編號與名稱**：第 6 節：Cluster Maintenance (叢集維護)
- **影片標題**：138. Backup and Restore Methods (備份與復原方法：ETCD)

## 2. 📌 核心概念摘要
etcd 是 Kubernetes 叢集的「大腦與唯一資料庫」，儲存了所有資源（Pod, Deployment, Secret 等）的狀態。本節的核心運作目標是：透過官方工具 `etcdctl` 搭配正確的 TLS 憑證，將當前的叢集狀態安全地匯出為快照檔 (Snapshot)，並在遭遇毀滅性災難時，能夠將快照還原 (Restore) 到全新的資料目錄中，讓叢集起死回生。

## 4. 🔑 知識點擷取 (Detailed Notes)
在操作 `etcdctl` 時，有幾個底層機制與考場規則必須嚴格遵守：

- **API 版本宣告 (極度重要)**：
  K8s 目前底層使用的都是 ETCD v3 版本的 API。但在某些 Linux 環境下，`etcdctl` 預設可能還停留在 v2。因此，執行任何指令前，強烈建議在指令開頭加上 `ETCDCTL_API=3`。

- **TLS 憑證認證 (Authentication)**：
  ETCD 是一個高度安全的資料庫，不允許任何人隨便存取。你必須在指令中帶入三個關鍵的憑證檔案，才能證明你的管理員身分（如影片截圖所示）：
  - `--cacert` (信任的憑證機構)
  - `--cert` (客戶端憑證)
  - `--key` (客戶端私鑰)

- **還原的底層邏輯 (Restore Logic)**：
  當你執行 `snapshot restore` 時，它並不會去覆蓋原本正在運行的 ETCD 資料庫。相反地，它會把快照檔解壓縮，產生一個「全新的資料目錄」。你必須手動去修改 ETCD 的靜態 Pod 設定檔 (`/etc/kubernetes/manifests/etcd.yaml`)，把 `data-dir` 的路徑指向這個新的目錄，Kubelet 偵測到檔案變更後，會自動重啟 ETCD 套用新資料。

## 5. 💻 CKA 必備實作指令 (Imperative Commands)
*(請將畫面中的指令與還原指令當作肌肉記憶練習)*

```bash
# 💡 考點 1：備份 ETCD (Snapshot Save)
# 考試時，若忘記憑證路徑，請去 cat /etc/kubernetes/manifests/etcd.yaml 裡面找
ETCDCTL_API=3 etcdctl snapshot save /opt/snapshot.db \
  --endpoints=https://127.0.0.1:2379 \
  --cacert=/etc/kubernetes/pki/etcd/ca.crt \
  --cert=/etc/kubernetes/pki/etcd/server.crt \
  --key=/etc/kubernetes/pki/etcd/server.key

# 💡 考點 2：驗證備份檔是否健康 (Snapshot Status)
ETCDCTL_API=3 etcdctl --write-out=table snapshot status /opt/snapshot.db

# 💡 考點 3：還原 ETCD (Snapshot Restore)
# 注意！這裡不需要帶入憑證，因為你是直接對「實體檔案」進行操作，解壓到 --data-dir 指定的新目錄
ETCDCTL_API=3 etcdctl snapshot restore /opt/snapshot.db \
  --data-dir=/var/lib/etcd-backup
```

## 6. 🚀 CKA 考試延伸與 Troubleshooting
🎯 **考試情境預測**：
> 「請將現有的 ETCD 叢集備份至 `/opt/backup/etcd.db`。接著，請將先前已準備好的快照檔 `/data/backup/snapshot-pre-boot.db` 還原至叢集中，並確保叢集能正常讀取新資料。」

🛑 **避坑指南 (學員最常踩的地雷)**：
> - **路徑打錯/找不到憑證**：考試時，每個環境的憑證路徑可能略有不同。千萬不要死背路徑！ 請直接開啟 `/etc/kubernetes/manifests/etcd.yaml`，在裡面尋找 `--trusted-ca-file`, `--cert-file`, `--key-file` 這三個參數後面接的路徑，直接複製貼上。
> - **忘記修改 Volumes**：在還原階段，修改 `etcd.yaml` 時，不僅要改 `command` 裡面的 `--data-dir`，檔案最下方的 `volumes` 中 `hostPath` 的 path 也必須一併改成你的新目錄 (例如 `/var/lib/etcd-backup`)。只要漏改一個，ETCD 就起不來。

🔧 **Troubleshooting (除錯方向)**：
> 如果你修改完 `etcd.yaml` 後，發現 `kubectl get nodes` 永遠報錯（連線被拒絕）。請第一時間去檢查你的 `etcd.yaml` 是不是有少打字或縮排錯誤！只要這個 YAML 有語法錯誤，Kubelet 就無法啟動這個 Static Pod，整個 API Server 就會癱瘓。可以使用 `crictl ps -a` 查看 etcd 容器是否有成功啟動。

---
> **💡 導師的隨堂測驗：**
> 在實務和考試中，尋找正確的憑證路徑是拿分關鍵。如果你登入了一台 Control Plane 節點，卻發現官方文件上寫的 `/etc/kubernetes/pki/etcd/...` 路徑底下找不到憑證檔案。
> 你知道該「查看哪一個設定檔」，才能百分之百確認目前正在運行的 ETCD，到底把憑證放在哪裡嗎？
