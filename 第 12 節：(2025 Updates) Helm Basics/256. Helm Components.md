# Helm 生態系基石：Repository 與 Artifact Hub (Helm Components)

## 📌 核心觀念摘要
* **雙層供應鏈架構**：為了跨越分散的開源社群，Helm 採用了「中央檢索、本地註冊」的設計。這能協助我們安全且高效地管理第三方應用軟體包 (Charts) 的來源，為企業級的自動化部署奠定供應鏈基礎。
* **Artifact Hub (網頁聚合器)**：就像是網購世界的 **Google 搜尋引擎或線上型錄**。它本身「不存放」任何實體貨物 (Charts)，只負責為全球各組織的倉庫進行統一編目、展示與導向。
* **Helm Repository (實體儲存庫)**：各家組織 (如 Bitnami, Nginx) 的 **實體發貨倉庫**。這是一個標準的伺服器，內部包含清單索引 (`index.yaml`) 以及實際打包好的 Chart 壓縮檔 (`.tgz`)。只有將這個倉庫註冊到本地端，Helm 才能真正載入該軟體。

## 💻 必考指令 (Imperative Commands)

在 CKA 考場中，熟練操作倉庫 (Repo) 是安裝任何第三方應用的前置通關連招：

```bash
# 1. 跨庫全域搜尋 (直接檢索 Artifact Hub)
# 當你不確定某軟體是由哪個組織託管時非常實用
helm search hub kube-prometheus-stack

# 2. 考場核心：新增指定的遠端 Helm 倉庫
# 語法: helm repo add <自定義倉庫別名> <倉庫網址>
helm repo add bitnami https://charts.bitnami.com/bitnami

# 3. 考場救命連招：同步遠端倉庫的最新索引至本地快照
# 🚨 致命細節：加完 Repo 後，不論題目有沒有要求，一律補上這條指令！
helm repo update

# 4. 本地倉庫精準搜尋：在已註冊的本地 Repo 快照中尋找
# 速度極快，且支援版本篩選過濾
helm search repo bitnami/nginx

# 5. 列出系統中已經註冊的所有倉庫清單
helm repo list
```

## 🛠️ 實戰與最佳實踐

> [!WARNING]
> **考試避坑：本地快照過期地雷**
> 如果考題指定要安裝「最新版」或「某特定次要版本」，而您先前已經加過該 Repo，請務必先手動執行 `helm repo update`！否則 `helm install` 會因為本地快照的版號與遠端不同步，無情噴出 `chart not found` 或 `version not found` 的致命錯誤。

> [!TIP]
> **SOP：考場安裝情境**
> 考題常會給定一個 URL（如：`https://kubernetes.github.io/ingress-nginx`），要求將倉庫命名為 `ingress-nginx` 並安裝指定版本。您的 SOP 應為：
> 1. `helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx`
> 2. `helm repo update` (即使題目沒說也必做)
> 3. `helm install my-ingress ingress-nginx/ingress-nginx --version 4.0.6 -n specific-namespace`

> [!CAUTION]
> **Troubleshooting 必殺技**
> - **`Error: no repositories to show`**：代表您尚未執行 `helm repo add` 來註冊任何倉庫，請先完成註冊。
> - **`Error: failed to download "..."`**：請檢查考場網路 (`curl -I <Repo-URL>`)；或是檢查倉庫別名是否拼錯（例如別名明明是 `bitnami`，搜尋或安裝時首碼就必須嚴格對齊 `bitnami/xxx`，可透過 `helm repo list` 快速核對）。

## 🧠 自我測驗

<details>
<summary>Q1: Artifact Hub 與 Helm Repository (實體倉庫) 最大的區別是什麼？</summary>

**解答：** 
Artifact Hub 是一個大型搜尋引擎與網頁型錄，它本身不儲存實體的套件檔案；而 Helm Repository 則是各家廠商（如 Bitnami, Nginx）實際託管存放 `index.yaml` 索引檔與 Chart 壓縮檔 (`.tgz`) 的實體伺服器。
</details>

<details>
<summary>Q2: 為什麼在安全隔離（Air-gapped）的企業地端環境中，不能依賴 Artifact Hub？</summary>

**解答：** 
因為 Artifact Hub 是公網資源，需要對外連線。在隔離環境中，企業必須自行架設私有的 Helm Repository（如 Harbor 或是 ChartMuseum），將通過資安審查的 Chart 放進該私有庫，然後讓內網的 Helm Client 去對這台私有伺服器執行 `helm repo add`。
</details>

<details>
<summary>Q3: 為什麼新增 Repo 後如果沒有立刻執行 helm repo update，常常會導致後續搜尋或安裝失敗？</summary>

**解答：** 
`helm repo add` 指令只是在本地端「記下」該倉庫的網址與別名。必須透過 `helm repo update` 指令，Helm 才會真正去遠端倉庫將完整的清單索引 (`index.yaml`) 抓取並快取到本地端。缺少這份最新的本地快照，系統就會不知道該去哪裡尋找對應的套件。
</details>
